# PowerShell 恶意脚本检测 v4.0 — 三层架构设计

> 基于与 Codex 对话达成的共识。2026-05-15。

---

## 核心思路

把 1593 个特征模式拆成三个处理层级，而不是让一个模型硬吃全部。

```
样本输入
    │
    ├── 满足 count≥30 且 posterior_entropy<0.1？
    │       │
    │       ├── YES → 【查表层】直接返回后验分布
    │       │
    │       └── NO  → 【归纳层】LGBM 模型预测
    │                    │
    │                    └── 测试集特有且模型不确定？
    │                            │
    │                            └── 【伪标签层】teacher 投票结果注入
    │
    └── 最终输出（查表层 + 归纳层 + 可选伪标签层融合）
```

---

## 一、查表层

### 规则
```python
posterior = (class_count + alpha * global_prior) / (count + alpha)
# alpha 建议从 5~10 起步 (Dirichlet shrinkage)

lookup_key = (
    count >= 30 AND
    entropy(posterior) < 0.1 AND
    posterior_max_prob >= 0.95
)
```

### 行为
- 满足条件的 key → 直接输出 `posterior` 作为预测概率
- 不满足条件的 key → 送入归纳层

### 覆盖估算
待实际统计。预计覆盖训练集 60-80% 的样本（高频确定模式占主导）。

---

## 二、归纳层

### 训练数据
- 只包含**不满足查表层条件**的样本
- 或者：所有样本，但对查表层样本降权重

### 模型
- 沿用 v3.0 的 KD 架构：
  - 4 Teacher (3 LGBM + 1 XGB) + SMOTE + StratifiedGroupKFold
  - 3 Student × 5 Fold × 3 Class LGBMRegressor
  - 软标签蒸馏（0.05×T1 + 0.95×T3）

### 特征
- 132 维（15 base + 105 pairwise + 12 统计特征）
- 与 v3.0 相同

---

## 三、伪标签层

### 适用条件
- **只对归纳层难区**做伪标签，不对查表层 key 做
- 目标 key 类型：
  - `global_unseen_key`: 整个 train 中完全未出现的特征组合
  - `weak_seen_key`: train 中出现过但 count 小、熵高、冲突明显

### 门控规则（严格）
```python
accept_pseudo_label = (
    3 个 teacher 类别一致 AND
    mean_confidence >= 0.97 AND
    margin (最大类 - 次大类) >= 0.25 AND
    teacher_pred_std < 0.05 AND
    key NOT in lookup_table  # 不污染查表层
)
```

### 注入方式
- 伪标签样本加入归纳层训练集
- 预估数量：1000-2000 条
- 宁少勿多

---

## 四、度量体系（替代随机 OOF）

### 两个 Canary 集合

**Canary-G (Group-Unseen)**
- 从 1593 个 key 中随机抽 200 个
- 这些 key 的**所有样本**永远不进 training
- 用途：测归纳层对完全未见模式的泛化

**Canary-S (Seen-Holdout)**
- 对高频 key（count ≥ 30），每个 key 留 20% 样本不进 training
- 用途：测查表层/模式先验对已见模式的稳定性

### 三个指标
| 指标 | 定义 | 含义 |
|------|------|------|
| Canary-G F1 | 只在 200 个 unseen key 上 | 未见模式泛化 |
| Canary-S F1 | 只在 seen holdout 上 | 已见模式稳定性 |
| Overall Canary | 按 test 中 seen/unseen 比例加权 | 综合"准平台"分 |

### OOF 的定位
- 不再作为主信号
- 只作为 sanity check（检测明显回归）

## 五、实验矩阵

| 实验 | 查表层 | 归纳层 | 伪标签层 | 预期 |
|------|--------|--------|----------|------|
| A | 无 | LGBM baseline | 无 | 基线 |
| B | count≥30 + 低熵 | LGBM (学剩余) | 无 | Canary-S↑, Canary-G持平 |
| C | count≥30 + 低熵 | LGBM (学剩余) | 0.97+投票 | Canary-G↑, Canary-S不降 |

### 判停逻辑

| 比较 | 判断 |
|------|------|
| B vs A: Canary-S↑, Canary-G不明显↓ | 查表层有效 ✅ |
| B vs A: Canary-G明显↓ | 查表太激进/归纳层变弱 ❌ |
| C vs B: Canary-G↑, Canary-S不↓ | 伪标签有效 ✅ |
| C vs B: 只提升OOF, Canary-G不涨 | 伪标签噪声/自强化 ❌ |

---

## 六、实现优先级

1. **先建度量体系**：Canary-G 200 key + Canary-S holdout → 拿到可信准平台分
2. **再拆查表层**：count≥30 + Dirichlet shrinkage + entropy → 验证 B vs A
3. **最后加伪标签**：严格门控 → 验证 C vs B
4. 任何一步判停失败 → 回退上一步

## 七、Dirichlet 先验收缩公式

```python
# global_prior: 全局类别分布 [P(0), P(1), P(2)]
# class_count: 该 key 在每个类上的样本数 [c0, c1, c2]
# alpha: 收缩强度 (建议 5~10)
# count = c0 + c1 + c2

posterior = (class_count + alpha * global_prior) / (count + alpha)

# 例如: 某 key 出现 3 次, [0,2,1], alpha=10
# posterior = ([0,2,1] + 10*[0.65,0.18,0.17]) / (3+10)
#           = ([0,2,1] + [6.5,1.8,1.7]) / 13
#           = [6.5, 3.8, 2.7] / 13
#           = [0.50, 0.29, 0.21]
# vs 经验频率 [0, 0.67, 0.33] → 明显更保守、更接近全局分布
```
