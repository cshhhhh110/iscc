> 2026-05-14，基于 v1.5~v3.0 全部迭代经验。

## 一、题目本质

- 15 个离散特征，每个 2-3 个取值（极低基数类别特征）
- 3 分类：0=正常，1=一般恶意，2=混淆恶意
- 指标：平台 Macro-F1
- 训练集 48065 条，测试集 20000 条

**关键约束：** 15 维特征高度离散且基数极小，所有可能取值组合只有 1593 种实际出现。这意味着大量样本共享完全相同的特征模式。这是本题一切挑战的根源。

## 二、核心矛盾：OOF 虚高 vs 平台塌陷

### 所有版本成绩一览

| 版本 | OOF | 平台 | gap | 方案 |
|------|-----|------|-----|------|
| v1.5 | 0.7555 | 0.6910 | **0.064** | 7候选 fusion |
| v1.6 | 0.7555 | 0.6884 | 0.067 | +交互特征/KMeans/SMOTE/Focal |
| v1.7 | 0.7554 | 0.6978 | 0.058 | +对抗验证 |
| v1.8 | 0.7519 | 0.6978 | 0.054 | tree+pseudo |
| v1.9 | 0.7554 | 0.6915 | 0.064 | fusion+pseudo |
| v2.0 | - | - | - | mega 100模型 未提交 |
| v3.0 | 0.7614 | 0.6940 | 0.067 | KD复现(抄歪) |
| **参考 0.713** | **0.7392** | **0.713** | **0.026** | KD+StratifiedGroup |

### 核心教训

**OOF 高不代表平台高。** OOF→平台 gap 从 0.026（参考）到 0.067（我们），差距 0.04。所有试图抬高 OOF 的操作（交互特征、对抗验证、SMOTE、Focal Loss、pseudo-label）都在平台面前失效了。唯一缩小 gap 的是 StratifiedGroupKFold——它让 OOF 变低了，但更接近平台真实水平。

## 三、什么有效、什么无效

### ✅ 有效的

| 方法 | 效果 | 原理 |
|------|------|------|
| **StratifiedGroupKFold by 特征key** | 缩小 gap 40% | 同模式不泄漏，OOF更真实 |
| **模式统计特征**（频率/冲突/分布） | +0.01~0.02 | 模型学到"这个模式可信度"而非死背类别 |
| **Knowledge Distillation** | +0.01~0.02 | 软标签平滑，防过拟合 |
| **Per-class 独立回归** | 稳定 | 避免类别间零和博弈 |
| **树模型（LGBM/XGB）** | 基线 | 15维类别数据最适合树模型 |
| **GPU 加速** | 提速 3-5x | CatBoost/XGBoost/LGBM 开 GPU |
| **特征pairwise交叉** (c1*10+c2) | +0.005 | 105 个二阶组合 |
| **SafeEmbedding** | 零风险 | 防 CUDA OOB 崩溃 |
| **teacher checkpoint** | 省时间 | 训完立刻存，崩了不白跑 |

### ❌ 无效的

| 方法 | 原因 |
|------|------|
| **交互特征** (product/ratio/diff) | 15维离散，乘法没意义 |
| **KMeans 聚类特征** | 离散数据上聚类无信息增益 |
| **对抗验证权重** | 训练/测试分布差异不大 |
| **Focal Loss** | 样本不是极度不平衡 |
| **NN 架构** (DCN/TabResNet/Transformer) | 15维低基数，embedding 过参数化 |
| **pseudo-label（低阈值）** | 8499条噪声反而带偏模型 |
| **100 模型简单平均** | 不加结构多样性，只加方差 |
| **融合权重暴力搜索** | 462×1377=63万组合，OOF过拟合 |
| **交互特征+KMeans+SMOTE 三件套** | 一起上反而互相干扰 |

### ⚠️ 有条件的

| 方法 | 条件 |
|------|------|
| **pseudo-label** | 阈值 0.95+，树模型，class2 不降门槛，3000-6000 条 |
| **SMOTE** | 只在 StratifiedGroupKFold 的每折内做 |
| **校准搜索** | 搜 bias 就够了，不要搜 temperature |
| **树模型** | 比 NN 好，但单模型不如多模型 KD |

## 四、0.713 参考方案完整拆解

### 训练流程

```
第 0 步（离线，代码未给出）：
  训练 4 个 Teacher (3 LGBM + 1 XGB)
  ├── SMOTE 过采样（每折内）
  ├── StratifiedGroupKFold by 特征key
  ├── 超参调优（→ "tuned"）
  └── 保存 4 个 .npz OOF 文件

第 1 步（train.py）：
  加载 4 个 .npz → 加权集成 → 温度软化 → 训练 student
  ├── Teacher 权重: 0.14273 / 0.12394 / 0.34562 / 0.38771
  ├── 软目标: 0.05×T1 + 0.95×T3（多温度蒸馏）
  ├── 3 Student × 5 Fold × 3 Class = 45 LGBMRegressor
  └── 模型存为 txt（booster.model_to_string()）极省空间

第 2 步（test.py）：
  加载 45 个 student → 平均 → bias 校正 → normalize → 提交
```

### 特征工程（131-132维）

```
15 维原始特征
+ 105 维 pairwise 交叉 (c1*10 + c2)
+ 频率特征 (pattern count, log1p)
+ 行级特征 (sum, nonzero count)
+ 冲突特征 (nunique per pattern, deterministic flag, ambiguous flag)
+ 分布特征 (per-pattern class fraction × 3, total count)
= 131~132 维
```

### 关键参数

```python
LGBM: n_estimators=300, lr=0.04, num_leaves=31, min_child_samples=30,
      subsample=0.8, colsample_bytree=0.8, reg_alpha=0.1, reg_lambda=1.0

XGB:  同上（n_estimators=300, lr=0.04, max_depth=6）

N_STUDENTS=3, N_FOLDS=5
Final bias: [1.0, 1.0791, 1.0007]
```

### 为什么 0.713 能以更低 OOF 拿到更高平台分

1. **StratifiedGroupKFold 让 OOF 诚实了**。同模式不切碎，OOF 评估的是模型对"没见过的新模式"的预测能力，而不是对"训练集里见过的模式"的记忆能力。

2. **KD 软标签比硬标签抗噪**。Teacher 的 0.7→0.2→0.1 比硬标签 1→0→0 更平滑，Student 不容易过拟合 noisy pattern。

3. **Per-class regression 避免了 softmax 的零和**。三个独立回归器各自判断"像不像"，最后 normalize，比一个三分类 softmax 更鲁棒。

4. **没有过度调参**。Teacher 权重是 4 个数，bias 是 3 个数，没有搜 63 万种组合。

### 传输方式
- 老机 `python -m http.server 8899 -d E:\赛题数据`
- 新机 `curl http://10.51.165.112:8899/xxx.zip -o E:\bisai\xxx.zip`

### 新机坑点
1. **系统 Python 3.8 抢 PATH** — pip/python 必须用完整路径
2. **SSH 无 GUI** — notepad 不可用，用 PowerShell 改文件
3. **中文路径在 SSH 里传参会截断** — 优先英文路径
4. **.cmd GBK 编码，.ps1 UTF-8**
5. **tar 默认解压到 C:\Users\20665，不一定是当前目录**

### GPU 加速三件套
```python
CatBoost:  task_type="GPU", devices="0"
XGBoost 3.x: device="cuda"  # 不是 tree_method="gpu_hist"
LightGBM: device="gpu"
```
加上 `n_jobs=4` / `thread_count=4` 替代原来的 `n_jobs=1`。

## 六、版本演进建议

### 如果让我从零再做一次

**第一版（1天）：**
直接用参考代码的架构：
- StratifiedGroupKFold by feature key
- 132 维特征（pairwise + 统计）
- 4 Teacher (SMOTE+LGBM/XGB) + 45 Student LGBMRegressor
- KD 软标签 + bias 搜索

**第二版（1天）：**
- 用参考的 .npz teacher OOF 训练 student（如果能拿到完整文件）
- 尝试不同的 KD 温度配比
- 搜索 teacher 权重

**第三版（1天）：**
- 尝试 pseudo-label（高质量，来自 KD ensemble）
- 微调超参（tree count, leaf num, reg）

### 不建议的方向
- 堆更多模型 → 边际收益递减
- NN/Transformer → 15 维类别数据上无用
- 复杂特征工程 → pairwise + 统计已经够了
- 暴力参数搜索 → 过拟合 OOF
