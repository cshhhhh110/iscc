# PowerShell 恶意脚本检测 — 完整总结

## 任务

- 15维低基数离散特征（每维 2-3 个取值）
- 三分类：0=正常脚本 1=一般恶意 2=混淆恶意
- 训练 48065 条，测试 20000 条
- 评估：Macro-F1

## 版本演进

### v1.0-v1.4（基础建设）
- ExtraTrees + HistGradientBoosting 双树融合
- 逐步加入 Target Encoding、DeepCrossNetwork、TabResidualNet、Pattern Lookup
- 扩展至 7 候选模型 + 单纯形权重融合
- OOF 从 0.753 提升到 0.7557

### v1.5（当前最佳基线）
- 7候选 fusion：tree/dcn/tab_resnet/pattern_lookup/catboost/xgboost/lgb
- 温度缩放 + class_bias 校准
- OOF 0.755537，**平台 0.69104**（最佳平台分）

### v1.6（特征工程尝试 — 无效）
- 交互特征：315个pairwise(乘积/差值/比值) → LightGBM筛选top-60
- KMeans 聚类距离特征（10维）
- SMOTE 过采样
- Focal Loss（gamma=2.0）
- 结果：树模型分数完全不变，NN 反而退步
- **结论：低基数离散特征产生的新特征仍是低基数离散量，无增量信息**

### v1.7（对抗验证 — 无效）
- LightGBM 判别器区分 train/test
- 对抗权重加权采样（1.5x oversample）
- 结果：树模型全线退步（tree 0.7551→0.7530）
- **结论：15维低基数特征无法表达 train→test 分布偏移，判别器 AUC≈0.5**

## 尝试过的方向及结果

| 方向 | 效果 | 原因 |
|------|------|------|
| Target Encoding | +0.002 | 类别统计平滑有效 |
| Pattern Lookup | +0.001 | 组合特征记忆有效 |
| DCN/TabResNet | ~0.754 | 嵌入+交叉层有效但不如树 |
| 7候选 Fusion | +0.0003 | 异构集成边际提升 |
| 交互特征 | 0 | 离散×离散=离散 |
| KMeans 距离 | 0 | 无法聚类离散点 |
| SMOTE | 0 | 合成样本无新信息 |
| Focal Loss | -0.0002 | gamma太大，标准CE更好 |
| 对抗验证 | -0.0015 | 判别器无法区分分布 |

## 模型架构（v1.5/v1.7 最终形态）

```
数据加载 → 15维离散特征
  ├─ ExtraTrees + HGB 双树融合 (target encoding + 温度校准)
  ├─ DeepCrossNetwork (embedding + cross layers)
  ├─ TabResidualNet (embedding + residual blocks)
  ├─ Pattern Lookup (精确组合记忆)
  ├─ CatBoost / XGBoost / LightGBM
  └─ 单纯形权重搜索 → Fusion
```

## 核心瓶颈

**OOF 0.755 → 平台 0.691，gap = 0.064**

15个特征全是 2-3 值的离散量。训练集和测试集的分布偏移存在，但无法用这些特征刻画。所有试图缩小 gap 的方法（特征增强、采样、对抗验证）都失败。

**0.69 很可能是这个数据集的天花板。**
