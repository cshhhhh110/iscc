# 版本归档工作流（跨项目复用）

## 工作流

每次完成一个版本迭代后，按以下步骤归档：

### 1. 归档文件（不复制模型）

```
版本记录/<version>/
  源码/           ← 复制 源码/*.py
  提交结果/        ← 复制 提交结果/submission_<version>.csv
  模型/           ← 只放 复现说明.md（模型文件留在当前工作区 模型/）
  README.md       ← 复制项目 README.md
  requirements.txt← 复制项目 requirements.txt
  ACTION_LOG.md   ← 复制项目级行动日志
  行动日志.md      ← 如果存在，一并复制
```

### 2. 写模型复现说明

在 `版本记录/<version>/模型/复现说明.md` 中记录：
- 当前版本所有模型文件清单（文件名、大小、用途）
- 配置文件清单
- 缓存文件清单
- 复现步骤（从源码 + 模型文件到生成 submission 的命令）

### 3. 更新日志

- 项目级 `ACTION_LOG.md`：追加 v<version> 条目（成绩、改进项、归档清单、教训）
- 根目录 `ACTION_LOG.md`：更新项目索引表、项目状态、执行记录
- 项目 `README.md`：更新成绩表

### 4. 模型文件保留

- 当前工作区 `模型/` 下的模型文件**不删除、不移动**
- 历史版本归档处只保留复现说明，避免大文件重复存储

## 执行脚本

```powershell
# archive_version.ps1 -ProjectPath <项目目录> -Version <vX.Y>
param($ProjectPath, $Version)

$verDir = "$ProjectPath\版本记录\$Version"
New-Item -ItemType Directory -Path "$verDir\源码", "$verDir\提交结果", "$verDir\模型" -Force

# 源码
Copy-Item "$ProjectPath\源码\*.py" "$verDir\源码\" -Force
# 结果
Copy-Item "$ProjectPath\提交结果\submission_$Version.csv" "$verDir\提交结果\" -Force
# 文档
Copy-Item "$ProjectPath\README.md", "$ProjectPath\requirements.txt", "$ProjectPath\ACTION_LOG.md" "$verDir\" -Force
if (Test-Path "$ProjectPath\行动日志.md") { Copy-Item "$ProjectPath\行动日志.md" "$verDir\" -Force }

# 模型复现说明（需手动补充文件清单）
"# $Version 模型文件清单（用于复现）`n`n待补充：模型文件列表和复现步骤" | Out-File "$verDir\模型\复现说明.md" -Encoding UTF8

Write-Host "归档完成: $verDir"
```
