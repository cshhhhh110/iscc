# 系统日志异常检测 — 优化计划

> 2026-05-08，基于环境实情和二进制漏洞检测题经验制定。

## 环境

| 项 | 实情 |
|------|------|
| GPU | NVIDIA RTX 3060 Laptop 6GB (CUDA 12.6) |
| PyTorch | 2.11.0+cu126 |
| XGBoost | 3.2.0 (CUDA + 稀疏矩阵) |
| LightGBM | 4.6.0 (pip 版不支持 CUDA，OpenCL 跑集显) |
| CatBoost | 1.2.10 |
| scikit-learn | 1.7.2 |

## 评分公式

```
Score = 0.15 × F1_detect + 0.50 × IoU_loc + 0.35 × F1_type
```

- F1_detect (15%): has_anomaly 二分类 Macro-F1
- IoU_loc (50%): 主异常区间 IoU 均值 — **最大杠杆**
- F1_type (35%): 主异常类型 10 类 Macro-F1

## v1.1 基线分析

| 指标 | 本地 OOF | 权重 |
|------|----------|------|
| F1_detect | 0.995 | 15% |
| IoU_loc | 0.848 | 50% |
| F1_type | 0.998 | 35% |
| **Score** | **0.9225** | — |
| **平台** | **0.826** | gap=0.096 |

问题：
1. SGDClassifier 概率极端化 → 10 个类型阈值全部饱和在 0.95
2. 解码器阈值搜索仅 24 候选
3. 早期停止指标与最终评分不对齐

## 优化路线

### v1.2 — XGBoost GPU 替换 LightGBM（当前）

- XGBoost 原生支持 CUDA + 稀疏 CSR 矩阵，可充分利用 3060
- 扩特征（charn 4 行级 + 2 文档级，hash 17→18 bits）
- 扩阈值搜索（24→350 候选，三阶段随机搜索）
- 目标：平台 >0.85

### v1.3 — 伪标签

- 用 v1.2 模型标注 test_data.csv (5000 条)
- 保守阈值（综合 doc 概率 + span 概率 + 类型概率）
- 高置信度样本加入训练
- 参考：二进制题伪标签 +0.022
- 目标：平台 >0.87

### v1.4 — BiLSTM+CRF 序列标注

- 用现有 HashingVectorizer 提取行级特征
- PyTorch BiLSTM + CRF 层直接预测精确区间边界
- 不需要后处理解码器，解决 IoU 瓶颈
- 目标：平台 >0.90

### v1.5 — 多种子集成

- 3 个不同 random_state 训练同一架构
- 预测概率平均
- 参考：+0.005~0.015
- 目标：平台 >0.91

## 来自二进制题的建议评估

| 建议 | 采纳 | 理由 |
|------|:---:|------|
| 伪标签 | 采纳 (v1.3) | 低风险高收益，二进制题验证过 |
| 多种子集成 | 采纳 (v1.5) | 零成本提分 |
| BiLSTM+CRF 序列标注 | 改造后采纳 (v1.4) | BERT 序列太长不可行，用轻量方案替代 |
| BERT+CRF | 不采纳 | 日志长度超 BERT 限制，GPU 内存紧张 |
| IoU-aware loss | 不适用当前 | 树模型不支持自定义 loss，v1.4 NN 阶段可考虑 |
| TTA | 不采纳 | 性价比低，推理慢 5-10 倍 |
