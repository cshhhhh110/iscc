
# 二进制文件漏洞检测 — 全面优化分析

## 当前状态

| 指标 | v1.3 分数 | 天花板判断 |
|------|------|------|
| Label F1 | 0.957 | 接近饱和，提升空间 <0.02 |
| **CWE Macro F1** | **0.830** | **主要瓶颈，目标 0.87+** |
| 综合 (README) | 84.9% | — |

CWE 是 86 类的长尾多分类，有一些类别只有个位数样本，训练严重不均衡。

## 现有架构

```
                    ┌─────────────────────┐
   binary file ─────▶│ 446 维表格特征       │──▶ VotingClassifier(HGB+ET)  ──┐
                    │ (PE 头/段/统计/API)  │    label 二分类                  │
                    └─────────────────────┘                                   │
                                                                              ├── 加权融合 ──▶ 最终预测
                    ┌─────────────────────┐                                   │
   binary file ─────▶│ 8192 字节窗口       │──▶ ByteMetaMultiTaskNet ─────────┘
                    │ embedding(24) →      │    (label + CWE 双头)
                    │ 4 层 conv → pool →   │
                    │ fusion MLP           │
                    └─────────────────────┘
```

**表格分支**: HGB + ExtraTrees 投票（label）；RF + ExtraTrees 投票（CWE），都用 soft voting  
**神经网络分支**: ByteMetaMultiTaskNet — 字节嵌层 (256→24) → 4 conv block → avg/max pool → 融合 MLP → label + CWE 双头  
**融合**: label 和 CWE 分别 41 点网格搜索最优加权系数  
**数据划分**: 80/20 train/val，含稀有 CWE 类别强制入训保证

## 根因分析

1. **CWE 长尾严重**：86 类中部分类只有个位数样本，`_split_indices` 能保证这些类进训练集，但模型在 1-2 个样本上学不到有意义的表征。`CrossEntropyLoss` 对多数类倾斜严重。

2. **表格 CWE 模型偏弱**：RF + ExtraTrees 的 VotingClassifier 对 86 类不均衡场景不是最优选择。CatBoost/LightGBM 的排序目标编码在这种场景通常更强。

3. **字节编码偏浅**：embedding=24 + 4 层 conv，最终特征 192 维。8192 字节窗口里包含函数序言、字符串表、导入表等结构化信息，浅层 conv 捕捉不到远距离依赖。

4. **融合过于粗糙**：单标量权重 × 概率，没有学到"表格擅长某几类 CWE、神经网络擅长另几类"的 per-class 策略。

5. **训练 epoch 偏少**：默认 10 epoch + patience=3，通常 3-5 轮就早停。学习率调度仅 ReduceLROnPlateau。

## 迭代方案（按优先级）

### Iter 1：Focal Loss（改一行，预期 +0.02-0.04）

**问题**：`CrossEntropyLoss` 把 86 个类平等对待，多数类梯度压倒少数类。

**方案**：把 CWE head 的 loss 换成 Focal Loss（gamma=2, alpha=class_weight）。Focal Loss 自动降低已分对样本的权重，迫使模型关注难分/少数的类。PyTorch 无内置但 15 行实现。

### Iter 2：CWE 表格模型换 CatBoost（预期 +0.01-0.03）

**问题**：`RandomForest(520) + ExtraTrees(720)` 投票对 86 类长尾效率低。

**方案**：替换为单个 `CatBoostClassifier`（iterations=600, depth=8, class_weights），6 万样本上训练 <2 分钟。CatBoost 内部用 ordered target encoding 处理类别不均衡优于 RF。

### Iter 3：Batch 内类别均衡采样（预期 +0.01-0.02）

**问题**：随机 shuffle → batch 中少数类经常缺位 → 梯度估计有偏。

**方案**：DataLoader 用 `WeightedRandomSampler`，权重 = 1/类别频数的平方根，确保每个 batch 都看到各类别。

### Iter 4：字节编码加深（预期 +0.01-0.02）

**问题**：8192 字节 → 4 层 conv → 192 维。感受野不够，高层次结构信息丢失。

**方案**：embedding 24→48，增加一或两层 conv，最后特征 256→384 维。或引入 1D 自注意力层捕捉长程依赖。

### Iter 5：融合升级 — MLP 元学习器（预期 +0.005-0.015）

**问题**：当前 `fused = w × neural + (1-w) × tree`，全局单一权重。

**方案**：把两个模型的 label 概率 + CWE 概率（共 1+86+1+86=174 维）拼起来，训一个 3 层 MLP 做最终分类。五折交叉防止过拟合。这样能学到 per-class 的信任策略。

### 兜底：数据增强

对稀有 CWE 类做 SMOTE 或简单的 copy-with-noise 扩充表格特征。

## 预期轨迹

```
v1.3 baseline:  CWE F1 = 0.830
  → Focal Loss:       0.850-0.855
  → +CatBoost:        0.855-0.865
  → +BalancedSample:  0.860-0.870
  → +DeeperEncoder:   0.865-0.875  ← 达到目标
```

## 实施顺序

1 → 2 → 3 → 评估 → 若未到 0.87 → 4 → 5

每个 iter 跑完整训练（epochs=15, batch_size=64），对比 `fused_cwe_macro_f1`。任一 iter 不升反降则回退，跳到下一个。
