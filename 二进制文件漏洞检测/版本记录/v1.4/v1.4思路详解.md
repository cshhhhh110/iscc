# v1.4 思路详解：双分支架构的奠基

v1.4 是二进制漏洞检测项目历史上最关键的单版本跳跃。平台 Macro-F1 从 0.849 跃升至 0.901，+0.052 的幅度在后续 16 个版本中从未被超越。这是因为 v1.4 不只是加了一个技巧，而是搭建了此后所有版本赖以演进的架构骨架。

## 前身 v1.3 的局限

v1.3 的方案非常朴素：从 PE 二进制文件中提取 446 维结构化特征——包括 PE 头字段、字节直方图、节表信息、DLL 和 API 导入、可打印字符串的威胁关键词——然后交给 LightGBM 做分类。这本质上是一个传统的特征工程 + GBDT 方案。

问题出在两个地方。第一个是代码层面的：阈值搜索逻辑有低级错误，导致二元分类的最优阈值被设在 0.08，意味着模型几乎把所有样本都判为漏洞。label 分布严重失衡，直接拉低了 CWE 阶段的起点。第二个是信息层面的：所有特征都是人工定义的结构化指标。PE 头的大小、段的熵、导入的 API 数量——这些有明确的语义，但要靠它们区分 CWE-121（栈溢出）和 CWE-122（堆溢出）几乎不可能，因为两者的 PE 结构在外观上没有本质区别。编译器指纹、字节序列模式、指令布局风格这些对漏洞分类至关重要的信号，完全不在 446 维特征的表征范围之内。

## 双分支互补

v1.4 的核心洞见是：让模型同时读到两种互补的信息源。

第一个分支沿用 v1.3 的 tabular 特征。这是人类可理解的结构化信号：文件有多大、导入了哪些 DLL、字符串里是否提到了 cmd.exe。LightGBM 特别擅长处理这种混合型表格数据，它会把 PE 头、导入表、字符串统计等信息学成一组可靠的基准判断。

第二个分支是全新的。v1.4 直接把二进制文件的原始字节喂给一个卷积神经网络。每个 PE 文件取 8192 个字节做 1D 卷积——这个长度足够覆盖 PE 头、导入表、部分代码段和一个文件的典型入口区域。网络用 5 层 Conv1D 逐级放大通道数，从 64 到 320，每一层有残差连接防止深层退化。最终输出两个头：一个做 label 的二分类，一个做 CWE 的 87 类分类，共享底层的字节特征提取器。

这两个分支的互补性非常强。LightGBM 能捕捉到的，CNN 看不太懂——比如 SECTION_ENTRY_EXECUTE 标志位在 PE 结构里的确切位置，CNN 的滑动窗口很难定位。反之，CNN 能捕捉到的，LightGBM 完全看不到——比如编译器在函数序言里总是插入的 push rbp / mov rbp, rsp 字节序列模式，以及某种特定的代码对齐填充字节。两种信号合并后构成了比各自独立丰富得多的信息空间。

最后的融合层把两个分支的输出拼在一起，过一个小型 MLP，联合预测 label 和 CWE。这种多任务学习的设计让 label 和 CWE 两个目标互相正则化——模型在学习"这是什么漏洞"的同时，必须同时回答"它是不是漏洞"，防止在某一方过拟合。

## 阈值 bug：一行代码的价值

v1.3 训练后要对验证集搜索最优的二元分类概率阈值。把 label 概率从 0 到 1 等间隔切分，每个阈值试一次 F1，取最高。

v1.3 的搜索范围和对结果的归一化处理有错误，导致黄金阈值被定为 0.08。这意味着概率超过 8% 就判漏洞。实际上大部分负样本的预测概率在 0.1-0.3 之间，全被误判为正。这个 bug 的影响远比看起来严重：由于 label=0 的样本全部被推入 CWE 路径，CWE 分类器被迫在噪声中预测，两步叠加让整体 Macro-F1 直接崩盘。

v1.4 修复就是让阈值搜索回归正常范围，最优阈值稳定在 0.45-0.55 区间。这一行修复估计单独贡献了 +0.03 以上的平台得分，也是软件开发中一个古老真理的例证：在高维 ML 场景下，一个 pipeline bug 的破坏力往往超过一个模型创新。

## 伪标签的铺设

v1.4 训练完成后没有直接把提交结果送交平台。而是用 v1.4 的 3-seed 集成模型对 test.csv 的全部 30,974 个样本做预测，筛选出高置信度的预测——label 概率超过 0.90，CWE 置信度也超过 0.90——作为"准标注"数据，与原始训练集合并。

这就是伪标签。17,343 条新增样本中，正向的大约占 45%，涵盖了大量稀有 CWE 类别在训练集中仅有个位数样本的情况。虽然 v1.4 本身还没有用上这批伪标签——它在等 v1.5 来继承——但这个策略的布局在 v1.4 就已经完成。

## 为什么 +0.052 是合理的

把 0.849 到 0.901 的飙升拆开，会发现它不是单因素的奇迹。阈值修复解决了 label 分布的严重偏差，是恢复性收益。双分支把信息源从一个变成两个，是结构性收益。多任务学习让两个预测目标互相约束，是正则化收益。三者叠加，0.052 是一个合理且可解释的跳跃。

更重要的是，v1.4 定义的架构——tabular + byte + 多任务融合——贯穿了后续全部 16 个版本直到 v3.0。之后所有的优化都是在不改变这个骨架的前提下，在特征层、训练策略、和推理后处理上做增量：加 Capstone 反汇编特征是扩宽 tabular 的信息面，换 CosineAnnealing 和 SWA 是让 CNN 训练更稳定，引入 N-gram 词典和元模型是在特征层注入领域知识而非改模型，都是在给 v1.4 画好的图纸添砖。
