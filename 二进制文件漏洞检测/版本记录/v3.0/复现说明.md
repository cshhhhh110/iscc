## 4. 复现步骤

### 一步到位

```bash
cd binary
python src/train.py --skip-pseudo
python src/test.py
```

train.py 内置自动构建 CWE N-gram 词典 (Phase 1.5)，无需手动预运行。

### 训练流程 (5 Phase)

| Phase | 内容 | 时间 (RTX 4060) |
|-------|------|------------------|
| 1 | 伪标签加载 | 即时 |
| 1.5 | CWE N-gram 词典自动构建 | ~30 分钟 |
| 2 | 特征提取 (首次) | ~4 小时 |
| 3 | 3-seed 训练 (GBDT + 神经网络) | ~1.5 小时 |
| 4 | 融合配置保存 | 即时 |
| 5 | 元模型训练 (OOF + N-gram) | ~5 分钟 |

### 核心配置 (fusion_config_v3.0.json)

| 参数 | 值 |
|------|-----|
| 融合模式 | scalar |
| Seeds | [42, 123, 202] |
| Neural CWE 权重 | 0.350 |
| Neural Label 权重 | 0.703 |
| 推理阈值 | 0.470 |
| Epochs | 30 |
| Focal Gamma | 1.5 |
| TTA 窗口 | 3 |
| 元模型 | LightGBM (num_leaves=15) |

## 5. 架构

```
PE Binary → 分支1: Tabular (935维, 含 N-gram 261维) → LightGBM
         → 分支2: Byte (8192窗口) → ByteMetaMultiTaskNet
         → 3-seed 加权融合 → scalar fusion
         → 元模型修正 (OOF预测 + N-gram特征) → submission
```

## 6. v2.4 → v3.0 关键改动

| 改动 | 影响 |
|------|------|
| CWE 字节 N-gram 词典 (261 维) | 类特字节模式注入，稀有类受益 |
| 元模型 (小型 LightGBM) | OOF 集成预测 + N-gram → CWE 二次修正 |
| 词典自动构建 | 免手动步骤，train.py 一键完成 |
| 路径适配 binary/data/models/output | 适配新目录结构 |
