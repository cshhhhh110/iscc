## 3. 复现步骤

### 3.1 准备数据
确保以下文件在项目根目录：
```
test.csv                    # 测试集
train.csv                   # 训练集
pseudo_train_v1.7.csv       # 伪标签（17,343 条）
binary-instructions/        # 二进制文件目录
```

### 3.2 训练（生成模型文件）
```bash
cd 二进制文件漏洞检测
python 源码/train.py
```

**训练耗时估算（RTX 4060 8GB）：**
- 特征提取（首次，含 Capstone 反汇编）：~4 小时（57K+ 样本）
- 单 seed 训练：~30 分钟（LightGBM + 30 epoch 神经网络）
- 3 seed 总计：~5.5 小时（首次含特征提取）

**如果已有特征缓存**（`train_features_v2.4.joblib` 等），特征提取会跳过，总时间约 1.5 小时。

### 3.3 预测
```bash
python 源码/test.py
```
输出：`提交结果/submission_v2.4.csv`

## 4. 架构概要

```
binary file
    ├── Tabular 分支 (674-dim)
    │   ├── PE 头特征 (59-dim)
    │   ├── 字节直方图 (256-dim)
    │   ├── 节特征 (22-dim) + DLL (20-dim) + API (63-dim) + 关键字 (26-dim)
    │   └── Capstone 反汇编 (228-dim)
    │       ├── 指令统计 (5-dim)
    │       ├── Opcode unigram (150-dim)
    │       ├── Opcode bigram (45-dim)
    │       ├── 寄存器使用 (26-dim)
    │       ├── 函数边界 (3-dim)
    │       ├── 操作数模式 (3-dim)
    │       └── 指令长度 (4-dim)
    │
    └── Byte 分支 (8192 字节窗口)
        └── 5 层 Conv1D (64→128→192→256→320)
            + residual connections
            + 双头输出 (label + CWE)

3-seed 集成 → 加权平均 → 阈值解码 → submission.csv
```
