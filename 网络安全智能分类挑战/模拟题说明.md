# 网络安全智能分类挑战模拟题

## 一、竞赛背景

随着大规模分布式系统在数据平台、云计算与智能运维中的广泛应用，日志已经成为监控系统状态、定位故障原因和支撑自动化运维的重要数据来源。传统日志分析方法往往只关注“是否异常”，难以满足真实生产环境中对异常区间定位和故障类型识别的需求。

本赛题聚焦于基于日志异常行为检测。参赛者需要利用机器学习或深度学习方法，对日志样本进行分析，并准确识别其所属的异常类别。竞赛旨在推动数据驱动的网络安全分析技术发展，探索更加高效、智能的日志异常检测方法。竞赛数据通过对原始系统日志进行提取与改写，形成日志异常检测数据。

竞赛旨在推动网络安全领域中的日志分析技术的发展，促进机器学习技术在网络安全场景中的应用，为构建更加安全可靠的网络环境提供技术支持。

## 二、任务说明

本赛题需要参赛队伍根据给出的训练数据（20000条系统日志信息及对应标签），训练一个系统日志异常检测模型，完成以下三类任务：

1. 异常检测（二分类）：判断测试集中的日志是否存在异常（`has_anomaly`：0/1）
2. 异常区间定位：定位主异常发生的起止位置（`primary_start_idx` / `primary_end_idx`）
3. 异常类型识别（多分类）：识别主异常的类型（共10大类）

可选类型：

`timeout_retry`、`resource_exhaustion`、`slow_burn_warning`、`state_conflict`、`parameter_drift`、`out_of_order`、`missing_step`、`duplicate_event`、`cross_component_mismatch`、`partial_recovery_loop`

## 三、数据说明

数据构成：由一系列系统日志文本记录构成。

训练集：共20000条日志记录，含日志文本与标签信息（文件：`train_data.csv`）

测试集：共5000条日志记录，仅含日志文本（文件：`test_data.csv`）

数据下载更新时间：2026年5月1日08:00

## 四、提交文件格式要求

最终提交 `.csv` 文件，使用 UTF-8 编码，必须包含表头，共6列：

`id`，`has_anomaly`，`primary_start_idx`，`primary_end_idx`，`primary_anomaly_type`，`all_spans`

字段说明：

`id`：日志记录编号（与测试集一一对应）

`has_anomaly`：是否存在异常，0表示无异常，1表示存在异常

`primary_start_idx`：主异常起始索引，无异常时填`-1`

`primary_end_idx`：主异常结束索引，无异常时填`-1`

`primary_anomaly_type`：主异常类型，无异常时填`none`，异常时从10种类型中选一个

`all_spans`：所有异常区间与类型，格式为`起始|结束|类型`，多异常用分号分隔，无异常留空

提交样例：

```csv
id,has_anomaly,primary_start_idx,primary_end_idx,primary_anomaly_type,all_spans
0,0,-1,-1,none,
1,1,12,16,timeout_retry,12|16|timeout_retry
2,1,5,9,state_conflict,5|9|state_conflict
3,1,8,10,missing_step,8|10|missing_step
4,0,-1,-1,none,
```

## 五、评分标准

最终联合评分由三部分加权构成：

`Score = 0.15 * F1_detect + 0.50 * IoU_loc + 0.35 * F1_type`

### 任务a：异常检测得分（权重0.15）

以`has_anomaly`为二分类标签，采用Macro-F1作为得分指标。

`F1_detect = 2 * (precision * recall) / (precision + recall)`

### 任务b：主异常区间定位得分（权重0.50）

仅对真实标签与提交结果同时判定为异常的样本计算。

指标为预测主异常区间与真实主异常区间的交并比（IoU）：

`IoU_loc = |[s_pred, e_pred] ∩ [s_gt, e_gt]| / |[s_pred, e_pred] ∪ [s_gt, e_gt]|`

所有满足条件样本的区间IoU取平均值作为该项得分，冗余或缺失区间都会扣分。

### 任务c：主异常类型识别得分（权重0.35）

仅对真实标签与提交结果同时判定为异常的样本计算。

统计`primary_anomaly_type`的预测结果，采用Macro-F1作为得分指标。

`F1_type = 2 * (precision * recall) / (precision + recall)`
