## 三、日常工作流（简化版）

```
cd E:\赛题数据

# 1. 同步
git pull

# 2. 改代码/训练
#    模型进 模型/ （不进git，已在.gitignore排除）
#    代码放 源码/

# 3. 提交
git add .
git commit -m "项目名 vX.Y: 做了什么"
git push
```

**提交消息格式**：`项目名 vX.Y: 关键改动`
- 例：`二进制 v2.4: self-attention + 稀有CWE增强`
- 例：`PowerShell v1.8: 新增pattern_lookup特征`

---

## 四、版本管理规则

| 目录 | 进 Git | 说明 |
|------|--------|------|
| `源码/` | 是 | 只放活跃版本代码 |
| `提交结果/` | 是 | 只放最新版本 |
| `模型/*.json` | 是 | 配置和报告（小文件） |
| `模型/*.pt` `.joblib` `.npy` | 否 | 二进制模型文件 |
| `版本记录/` | 否 | 历史归档，仅本地 |
| `binaries/` | 否 | 原始数据 |
| `docker容器/` | 否 | Docker材料 |
| 大CSV数据 | 否 | train.csv/test.csv |

---

## 五、新增版本时的完整流程

```
1. 训练完成 → 模型进 模型/model_bundle_vX.Y.xxx
2. 预测完成 → 结果进 提交结果/submission_vX.Y.csv
3. 复制当前 源码/ 模型/ 提交结果/ → 版本记录/vX.Y/
4. 更新项目 ACTION_LOG.md
5. 更新总目录 ACTION_LOG.md
6. git add . && git commit -m "项目 vX.Y: 做了什么" && git push
```

---

## 六、关键经验

- 二进制：fusion_mode 必须 scalar，不能 mlp；伪标签阈值 0.9，不能 0.7
- 训练崩溃后清理残留进程
- 长任务完成立刻通知用户
- 特征缓存跨版本可复用
